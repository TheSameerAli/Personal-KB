# hack the kitty 2026 — Virtual Kickoff Main Event Plan

## Short Verdict

The kickoff should feel like a **live launch show**, not an admin meeting.
Because this is a **virtual Discord-first** event, the structure should optimise for:

1. **Fast orientation** — what this is, how it works, and what to do next
2. **Energy** — make the start feel like a moment people want to be part of
3. **Action** — get people into `#hackathon-2026`, `#ideas`, `#help`, team formation, and building immediately
4. **Clarity** — remove confusion around rules, submissions, judging, and timing

## Recommended Event Shape

**Total length:** 60 minutes

### Segment 1 — Pre-show lobby
**Time:** 11:45–12:00 BST
**Owner:** [[Ruhi]] + mods
**Goal:** reduce cold-start awkwardness and get people settled before the official start

**What happens**
- Open Discord stage/voice early
- Loop a holding slide / pinned message with:
  - start time
  - event agenda
  - important channels
  - website link
  - submission basics
- Music or light banter if that fits the brand
- Mods welcome people in chat and direct them to:
  - `#hackathon-2026`
  - `#ideas`
  - `#help`
- Ask people in chat:
  - where they’re joining from
  - solo or team
  - what they want to build

**Why this matters**
For a virtual event, the pre-show creates social proof and activity before the actual kickoff starts.

### Segment 2 — Opening + hype
**Time:** 12:00–12:05
**Speaker:** [[Ruhi]]
**Goal:** make the event feel live, real, and exciting

**Cover**
- Welcome everyone to `#hackthekitty`
- Thank them for showing up
- Quick one-line framing of the community vibe
- Hand over to Sameer as the host/founder voice

**Tone**
Warm, energetic, slightly playful, internet-native.

### Segment 3 — Founder welcome / why this exists
**Time:** 12:05–12:12
**Speaker:** [[Sameer Ali]]
**Goal:** give the event emotional meaning, not just logistics

**Cover**
- Why [[coding.kitty]] is running this
- Why a cat-themed hackathon is fun *and* worth building for
- The spirit of the event:
  - beginner-friendly
  - creative
  - internet-native
  - community-first
- Set expectation that success is not only winning; it is also shipping, learning, and meeting people

**Key message**
This should make people feel: “I want to be part of this.”

---

### Segment 4 — Theme reveal / interpretation
**Time:** 12:12–12:20
**Speaker:** [[Sameer Ali]]
**Goal:** make the theme usable, not just memorable

**Theme:** **World Cat Domination Day**

**Frame it like this**
- Cats already run the internet
- Now participants are building things that help cats “take over” in a positive, useful, wholesome way

**Give concrete interpretation buckets**
- **For cats** — adoption, lost/found, health, feeding, welfare
- **For cat owners** — reminders, behaviour tools, expense tools, community
- **For the community** — shelters, rescue workflows, volunteering, TNR, fundraising
- **For developers** — cat APIs, cat-themed tools, open-source utilities

**Important clarification**
Say clearly that the theme is broad on purpose; creative interpretations are welcome as long as the project meaningfully connects to cats / the cat community and has positive impact.

### Segment 5 — How the hackathon works
**Time:** 12:20–12:28
**Speaker:** [[Ruhi]]
**Goal:** remove operational confusion early

**Cover**
- Dates: **24 June → 7 July, submissions due 22:00 BST**
- Format: **fully virtual**
- Team size: **solo or pair only**
- Registration via **Discord sign-in**
- Teams: register solo first, invite partner later if needed
- Where participants should spend time:
  - `#hackathon-2026` for announcements
  - `#ideas` for brainstorming
  - `#help` for support
  - `Social` voice for hanging out
  - `Office Hours` voice for help/mentorship

**Key outcome**
By the end of this section, nobody should be unsure where to go or how to participate.

---

### Segment 6 — Rules, judging, and submission
**Time:** 12:28–12:38
**Speaker:** [[Ruhi]] with optional Sameer emphasis
**Goal:** stop the FAQs before they happen

**Must-cover points**
- Build during the hackathon period; no old projects re-skinned for the theme
- AI tools are allowed, but originality still matters
- Any tech stack is allowed
- Submission includes:
  1. ZIP file
  2. short video demo
  3. GitHub repo link
- For pairs, the **team leader** submits
- Judging is **anonymous** by Reference ID
- Six judging categories:
  - technical
  - innovation
  - theme relevance
  - documentation
  - security
  - UX/UI

**Recommended emphasis**
Tell participants that the best projects are not necessarily the biggest; they are usually the clearest, most complete, and easiest to demo.

---

### Segment 7 — What a strong project looks like
**Time:** 12:38–12:45
**Speaker:** [[Sameer Ali]]
**Goal:** improve project quality and completion rate

**Give practical advice**
- Start with a small, demoable scope
- Build the core experience first
- Choose an idea that can be shown in 2–3 minutes
- Use the theme clearly instead of vaguely
- Keep docs clean
- Don’t leave the demo video until the last day
- Ask for help early in `#help`

**Useful framing**
“How to not die at a hackathon” works well here.

---

### Segment 8 — Team formation + first actions
**Time:** 12:45–12:53
**Speaker:** [[Ruhi]]
**Goal:** convert passive listeners into active participants immediately

**Do not just say “go network.”**
Instead, give a concrete post-event flow:

1. If you are solo and want a teammate, go to the Team Finder / `#ideas`
2. Post in one message:
   - name
   - timezone
   - skills
   - idea or interest area
3. If you already have a team, decide your project before today ends
4. If stuck, ask in `#help` or join `Social` / `Office Hours`

**Optional interactive moment**
Take 3–5 rapid-fire project ideas from chat to create momentum.

---

### Segment 9 — Q&A
**Time:** 12:53–12:58
**Speakers:** [[Sameer Ali]] + [[Ruhi]]
**Goal:** resolve live blockers while energy is still high

**Best practice**
- Take questions from chat, not open-mic chaos
- Prioritise questions about eligibility, theme fit, team rules, and submission
- Mods cluster duplicates and feed the best ones to speakers

---

### Segment 10 — Launch moment
**Time:** 12:58–13:00
**Speaker:** [[Sameer Ali]]
**Goal:** end with momentum and clarity

**Final CTA should include**
- the hackathon is officially live
- where to go next
- what to do in the next 15 minutes
- reminder of submission deadline

**Example ending**
- Join `#hackathon-2026` for updates
- Post in `#ideas` if you need a project or teammate
- Use `#help` when blocked
- Get your project chosen today
- Go build something cool for the cats

## Production Checklist for a Virtual Discord Event

### Before the event
- Confirm Discord `#main-event` setup works
- Confirm stage/voice permissions
- Prepare holding slide / pinned links post
- Prepare moderator run sheet
- Prepare FAQ answers in a private organiser note
- Confirm who is watching chat
- Confirm fallback if audio breaks
- Confirm recording policy if applicable

### During the event
- One person moderates chat
- One person handles speaker handoffs
- One person watches time
- One person collects and filters questions

### After the event
- Post recap in `#hackathon-2026`
- Repost all important links
- Push team-formation prompt
- Keep `#help` actively staffed for the first few hours after kickoff

## Biggest Risks for This Specific Kickoff

### 1. Too much admin, not enough energy
If it feels like a rules meeting, people tab out.

### 2. Theme sounds fun but not practical
Fix by giving plenty of concrete examples.

### 3. No structured post-event action
If people leave without a next step, momentum drops fast.

### 4. Virtual awkwardness
Use chat prompts, pinned actions, and moderated Q&A to keep things moving.

### 5. Beginners feel intimidated
Explicitly say all skill levels are welcome and that smaller complete projects can score well.

## Best-Fit Recommendation

For this event, the strongest structure is:

- **5 mins** pre-show warmup
- **5 mins** welcome + hype
- **8 mins** Sameer story / why this exists
- **8 mins** theme breakdown
- **8 mins** how it works
- **10 mins** rules + judging + submissions
- **7 mins** what makes a great project
- **8 mins** team formation + next actions
- **5 mins** Q&A
- **2 mins** launch close

This gives the kickoff the right balance of:
- excitement
- orientation
- community energy
- operational clarity
