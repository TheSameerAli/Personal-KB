## Theme — World Cat Domination Day

> *The cats are taking over. Your job? Help them do it.*

Build a project that drives **positive impact** for cats, cat owners, and the broader cat community. Think world domination — but wholesome.

> [!TIP] What counts?
> If it makes the world better for cats (and the humans who serve them), it counts. Creative interpretations are welcome.

### Project Inspiration

**For the cats**
- Adoption platforms & lost cat finders
- Health trackers & feeding schedulers
- Anything that makes a cat's life more comfortable

**For cat owners**
- Vet appointment reminders & behaviour decoders
- Expense trackers & community forums
- Tools that help humans be better cat servants

**For the community**
- Shelter volunteer coordination
- TNR (trap-neuter-return) mapping
- Fundraising tools & rescue organisation platforms

**For developers**
- Cat-themed dev tools & APIs for cat data
- Open-source libraries with cat puns in the docs
- Anything that makes coding more feline-friendly

## Judging Criteria

Projects are scored **anonymously** using only the Reference ID.

| Category | Weight | What They're Looking For |
|---|---|---|
| Technical Execution | 25% | Code quality, architecture, functionality |
| Innovation & Creativity | 20% | Originality and creative problem-solving |
| Security | 15% | Secure coding + responsible data handling |
| UX / UI | 15% | User experience and interface design |
| Documentation | 15% | Clear README, setup instructions, explanation |
| Theme Relevance | 10% | How well it ties into the hackathon theme |

> [!TIP] Bonus Points
> Include an [[Aikido Security]] scan report for bonus points on the Security category.

## Submission Requirements

Submit all three of the following:

1. **ZIP file** of your project — max **100 MB**
2. **Short video demo**
3. **GitHub repo link**

> [!INFO] Who submits?
> Solo participants submit from their own dashboard. For pairs, the **team leader** handles submission.

## Rules

### Project Rules
- All code must be written during the hackathon: **June 24 – July 7**
- Planning and brainstorming beforehand is fine — no pre-written code
- Project must relate to the theme and support **positive impact**
- No pre-existing projects rebranded with cat emojis
- Any tech stack allowed: web, mobile, CLI, hardware, etc.
- AI tools are allowed — but original thinking and effort must be evident
- No plagiarism, cheating, or recycled projects

### Behaviour Rules
- Be respectful to participants, judges, and organisers
