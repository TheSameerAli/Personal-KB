# Curriculum Plan

## Curriculum Model

The curriculum should be modular and project-first.

Instead of forcing every learner through the same fixed sequence, coding.kitty/learn should generate a path from the learner's app idea.

Each path maps to:

- project archetype
- recommended tool route
- required concepts
- optional concepts
- build sprints
- prompt exercises
- review checkpoints
- Git checkpoints
- deployment steps

## Backbone Path

### Phase 1: Builder Mindset

- what software is
- what AI coding agents can and cannot do
- how apps, websites, APIs, databases, and hosting fit together
- how to describe an app idea clearly
- how to break an idea into features

### Phase 2: First App

Recommended beginner archetypes:

- personal website
- simple productivity app
- content/blog app
- simple dashboard
- lightweight AI wrapper

### Phase 3: Prompting Agents

- clear build prompts
- asking for a plan
- asking the agent to explain code
- asking for small scoped changes
- reviewing AI output
- debugging with AI

### Phase 4: Software Basics

- files and folders
- HTML, CSS, and JavaScript basics
- APIs
- databases
- authentication
- environment variables
- terminal basics
- dependencies and package managers
- common error messages

### Phase 5: Version Control

Teach version control as:

> Save points for your project.

Learners should understand:

- Git vs GitHub
- repositories
- commits
- branches at a beginner level
- reading diffs
- recovering from mistakes
- asking an AI agent to explain changes before committing

### Phase 6: Infrastructure and Deployment

Teach infrastructure as:

> Where your app lives and what it depends on after it leaves your laptop or browser.

Learners should understand:

- deployment
- static sites vs full-stack apps
- hosting providers
- domains and DNS
- environment variables
- databases in production
- logs and common deployment failures
- costs, limits, and security basics

### Phase 7: Ship and Improve

- deploy the app
- test the live version
- fix bugs
- collect feedback
- make small improvements
- share the project

## First Project Tracks

### Personal Website

Best for:

- first shipped project
- free path
- creators, students, and professionals

Concepts:

- HTML/CSS basics
- page structure
- responsive design
- version control checkpoints
- static deployment
- domains

### Simple Productivity App

Best for:

- paid or deeper path
- professionals automating work
- proving the personalized-course promise

Concepts:

- JavaScript behavior
- forms
- state
- data persistence
- optional database
- optional authentication
- deployment with backend services
