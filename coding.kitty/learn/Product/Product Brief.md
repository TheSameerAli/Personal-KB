# Product Brief

## Product Definition

coding.kitty/learn is an AI Builder Learning Platform for non-technical people who want to turn an app idea into a shipped project.

The product should feel like an interactive build companion, not a passive course library.

## Core Promise

> Turn your app idea into a personalized AI-building course, then ship it with enough software understanding to own the result.

## User Journey

1. Learner describes an app idea.
2. Platform asks a short set of clarifying questions.
3. Platform maps the idea to a supported project archetype.
4. Platform recommends a tool route.
5. Platform generates a course map from curated learning blocks.
6. Learner completes guided build sprints using external AI coding tools.
7. Learner reviews changes, saves checkpoints, deploys, and improves the app.

## MVP Goal

Prove that a non-technical learner can describe an idea, receive a credible personalized build course, and use that course to build, understand, version, and deploy a simple real project with AI assistance.

## MVP Included

- coding.kitty/learn identity
- idea intake
- project diagnosis
- project archetype matching
- personalized course map
- guided task checklist
- prompt templates
- tool-agnostic explanations
- 2-3 beginner-friendly tool recipes
- beginner software fundamentals
- GitHub/version control checkpoints
- deployment walkthrough
- basic infrastructure explainer
- completion page or project showcase

## Supported MVP Archetypes

Start with:

- personal website
- simple productivity app
- content/blog app
- simple dashboard
- lightweight AI wrapper

Default free path:

> Personal website.

Default deeper or paid path:

> Simple productivity app.

## Not Included Yet

- full in-browser IDE
- custom coding agent execution
- unbounded course generation for any app idea
- complex backend projects beyond supported archetypes
- full certificate system
- cohort operations
- large template marketplace
- advanced community system

## Platform Components

- app idea intake
- archetype matcher
- route selector
- course map
- lesson/task page
- prompt builder
- project checklist
- progress tracking
- checkpoint reminders
- version control guidance
- deployment guidance
- project completion/share page

## Feature Backlog

### Now

- constrained idea intake
- course map generation
- personal website path
- prompt templates
- Git as save points
- static deployment walkthrough
- progress checklist

### Next

- simple productivity app path
- local storage and database basics
- deeper debugging drills
- project showcase
- Discord/community handoff
- learner feedback capture

### Later

- more archetypes
- sponsor-backed labs
- office hours and project reviews
- certificates or badges
- team/school licensing
- built-in coding workspace

## Product Principles

- Build first, explain when needed.
- Personalize around the learner's idea.
- Use AI to assemble and adapt the path, but keep teaching grounded in curated content.
- Teach learners how to guide AI, not blindly trust it.
- Make version control and deployment core skills.
- Keep the experience friendly, practical, and credible.
