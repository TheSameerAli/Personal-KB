# Business and GTM

## Business Thesis

AI coding tools are making software creation accessible to non-technical people, but many learners still need structure, confidence, and practical software literacy.

coding.kitty/learn can become the trusted beginner-friendly bridge between idea and shipped app.

Sharper thesis:

> People do not only want to learn "vibe coding." They want to build their own idea without getting lost.

## Positioning

Bring your app idea. coding.kitty/learn turns it into a personalized AI-building course and helps you ship it, even if you are starting from zero.

Short message:

> Your idea becomes the course.

Use "vibe coding" as the cultural hook, but position the durable category as:

> AI Builder Learning Platform.

## Revenue Model

### Free

- idea intake
- simple personalized course map
- one beginner archetype path
- basic prompt templates
- basic progress tracking
- deployment checklist
- community entry, if useful

Purpose:

- build trust
- prove the learning format
- create shareable learner outputs
- convert the Coding Kitty audience into platform users

### Paid

- deeper personalized project paths
- productivity app path
- advanced archetypes
- deeper Git/GitHub workflow
- deployment/infrastructure modules
- templates
- project reviews
- office hours
- challenges or cohorts

### Sponsor-Supported

- "Build with X" labs
- tool-specific modules
- sponsored build challenges
- partner resource pages
- hackathon tie-ins
- education campaigns for developer tools

Sponsor fit:

- AI coding tools
- cloud platforms
- deployment platforms
- database providers
- security tools
- observability tools
- Git/GitHub ecosystem tools

## Brand Guidance

`coding.kitty` remains the media and education brand.

`coding.kitty/learn` is the interactive learning platform underneath it.

Tone:

- friendly
- practical
- encouraging
- playful but credible
- beginner-safe

Theme level:

- medium Coding Kitty flavor
- mascot guidance is okay
- playful lesson names are okay
- the learning interface should still feel credible for professionals

## Pricing Questions

- One-time paid project path, subscription, or hybrid?
- Should sponsors subsidize free content?
- What makes a learner upgrade after the free path?
- Is the paid value generated path quality, guided workspace, review/support, or complex archetypes?
- Should team or school licenses exist later?
