# coding.kitty/learn - Executive Brief

## One-Line Concept

coding.kitty/learn turns a learner's app idea into a personalized AI-building course, then helps them build, understand, version, deploy, and improve it.

## Product Thesis

AI coding tools make software creation more accessible, but they do not automatically give beginners software judgment.

coding.kitty/learn should occupy the space between:

- fixed vibe-coding courses
- AI app builders
- traditional syntax-first coding education

The product should help learners ship real projects while learning the concepts that let them stay in control.

## Core Promise

> Go from idea to deployed app using AI coding agents, while learning the software basics needed to own the result.

## Audience

Primary learners:

- non-technical people
- creators
- students
- founders and operators
- professionals trying to automate work
- curious beginners intimidated by traditional coding

They are not necessarily trying to become software engineers on day one. They want agency with software.

## Differentiation

Most courses start with a fixed syllabus. Most app builders start with a prompt box.

coding.kitty/learn starts with:

> What do you want to build?

Then it maps the answer to:

- project archetype
- tool route
- required concepts
- optional concepts
- guided build sprints
- prompt templates
- review checkpoints
- Git checkpoints
- deployment steps

## MVP Shape

The MVP should not support every possible app idea.

Start with 3-5 constrained archetypes:

- personal website
- simple productivity app
- content/blog app
- simple dashboard
- lightweight AI wrapper

The product can personalize examples, prompts, wording, and tool route inside these archetypes without relying on fully unbounded AI curriculum generation.

## First Wedge

Free path:

> Build and deploy your first personal website with AI.

Why this works:

- easy to finish
- visually rewarding
- useful across learner groups
- naturally teaches HTML/CSS, GitHub, deployment, domains, and prompting

Paid/deeper path:

> Build a simple productivity app with an AI coding agent.

Why this works:

- clearer willingness to pay
- introduces data, persistence, forms, workflows, databases, and more infrastructure

## Product Model

The platform should behave like a guided course assembler:

```text
Learner idea
  -> project diagnosis
  -> archetype match
  -> tool route
  -> course map
  -> build sprints
  -> prompts, checklists, checkpoints, deployment
```

The AI layer acts as planner and tutor over a curated block library. It should personalize and sequence, not invent all teaching from scratch.

## Next Decision

Choose the first MVP path:

- personal website first, because it is easier to finish and share
- simple productivity app first, because it proves the bespoke-course promise more strongly
- both, with personal website as the free path and productivity app as the paid/deeper path
