# MVP Scope

## MVP Learning Path

> Turn your app idea into a guided AI-building course.

MVP constraint:

The first version should support a small set of project archetypes rather than every possible idea.

## MVP Goal

Prove that a non-technical learner can describe an app idea, receive a credible personalized build course, and use that course to build, understand, version, and deploy a simple real project with AI assistance.

## Included

- coding.kitty/learn identity
- idea intake
- project diagnosis
- personalized course map
- structured lesson path assembled from reusable blocks
- interactive task checklist
- prompt templates
- tool-agnostic guidance
- support for 2-3 beginner-friendly tools
- basic software fundamentals
- beginner-friendly version control flow
- deployment walkthrough
- simple infrastructure explainer
- completion page or project showcase

## Supported MVP Archetypes

Start with 3-5 archetypes:

- personal website
- simple productivity app
- content/blog app
- simple dashboard
- lightweight AI wrapper

The platform can personalize language, examples, prompts, and feature order inside those archetypes.

## Not Included Yet

- full in-browser IDE
- custom AI agent execution
- unbounded course generation for any possible app idea
- complex backend projects beyond supported archetypes
- paid cohort operations
- advanced certificates
- large marketplace of templates

## MVP Questions

- How constrained should the idea intake be in v1?
- Should the first MVP default to personal website, or should it immediately support a simple productivity app because that better proves the bespoke-course promise?
- Should the first version guide users through external tools, or include a lightweight built-in workspace?
- Which tools should be supported first?
- Should project sharing/showcase be included in MVP?
- What should be free versus paid?
