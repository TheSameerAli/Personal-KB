# Infrastructure and Deployment Module

## Learning Goal

Learners should understand enough infrastructure to know where their app lives, how it runs, and what can break after deployment.

## Topics

- what deployment means
- static sites vs full-stack apps
- hosting providers
- domains
- DNS basics
- environment variables
- databases in production
- APIs and backend services
- authentication services
- logs and error monitoring
- build steps
- CI/CD at a beginner level
- common deployment failures
- costs, limits, and security basics

## Teaching Angle

Infrastructure should be taught through real deployment tasks, not as abstract cloud theory.

The learner should be able to answer:

- Where is my app hosted?
- Where does the data live?
- What services does my app depend on?
- What happens when I push a change?
- How do I know when something breaks?
